HangFire

A simple way to kick off long-running processes from the ASP.NET request processing pipeline, backed by SQL Server or Redis.

Install via NuGet

Simple

HangFire is as simple as the use of Thread Pool to perform your fire-and-forget tasks:

BackgroundJob.Enqueue(
    () => Console.WriteLine("Hello!"));

Reliable

HangFire uses a reliable fetching algorithm together with the automatic job retry logic.

When a background task was created, it is guaranteed that it will be performed at least once, even if the host process has been terminated.

Designed for ASP.NET

Background jobs are being saved into a persistent storage and can survive the application pool recycling process.

If the performance of a job was terminated due to ASP.NET shutdown timeout, it will be retried after the restart of an application.

Efficient

Job processing time is virtually independent of their count – worst job processing algorithm complexity is O(log(n)).

Transparent

You can track the detailed state of each job and see the statistics using the integrated web interface.

Scalable

You can run multiple instances of HangFire Server, either inside an ASP.NET application or outside of it, as a console application or Windows Service.

Extensible

You can extend HangFire using different methods: write filters for the job creation/processing pipeline, create new states or implement support for your favourite storage.

Self-maintainable

HangFire keeps your storage as clean as possible and removes old records. It will not become bloat up with time.

Open source

HangFire is open source software and is completely free for commercial use. It is licensed under LGPLv3 license.
